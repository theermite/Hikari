## Détail des 5 assertions PET (verbatim)

Toutes les 5 sont dans `src-tauri/crates/automation/tests/engine.rs`, contre l'API
publique du crate :

1. `should_run_action_when_condition_true` — un automation actif, sans condition
   bloquante, retourne `Decision::Run(actions)`.
2. `should_refuse_when_condition_unevaluable` — une `VariableEquals` sur une variable
   absente du contexte retourne `Decision::Refused(ConditionUnevaluable{..})`, jamais
   un run silencieux.
3. `should_reject_cycle_at_save` — deux automations qui se rappellent l'une l'autre
   via `RunAutomation` sont refusées **à l'enregistrement** (`EngineError::
   CycleDetected`), avant tout risque d'exécution. Couvre aussi l'auto-référence
   directe (`should_reject_self_referencing_cycle_at_save`).
4. `should_reject_unknown_trigger_type` — un JSON avec `"trigger":{"type":"webhook"}`
   (type inconnu, hors énum fermée) échoue à la désérialisation
   (`EngineError::InvalidJson`), jamais silencieusement ignoré.
5. `should_never_expose_event_trigger_as_deck_key` — `deck_eligible_automations()` ne
   retourne que les automations à déclencheur `Button`; un déclencheur `Event`,
   `ChatCommand` ou `Timer` n'apparaît jamais dans la liste.
