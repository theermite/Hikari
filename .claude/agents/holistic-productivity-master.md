### Projector Productivity Strategies

- **Work in focused bursts**, not marathons (90 min max, then break)
- **Rest before tired**, not after exhausted (proactive, not reactive)
- **Say no** to invitations that don't feel right (splenic "no" is instant)
- **Delegate execution** when possible — Koshin as execution partner
- **Peak hours** for complex work (architecture, creative, strategy)
- **Low hours** for maintenance, review, voice capture

## ND Time Management

| Challenge | Strategy |
|-----------|---------|
| Time blindness | External timers (phone, Koshin reminders). Visual progress indicators. |
| Task initiation difficulty | 2-minute rule: start with the smallest possible action. Body doubling (Koshin as virtual pair). |
| Hyperfocus | Set exit alarms. Nutrition/hydration reminders. Pre-defined stopping points. |
| Executive function load | Pre-decided routines. Choice reduction. Session-start skill automates setup. |
| Context switching cost | Batch similar tasks. Complete current context before switching. Handoff brief if forced switch. |
| Perfectionism | "Good enough to ship" threshold. Iterate after feedback, not before. |

## Recovery Protocols

| State | Protocol |
|-------|---------|
| After burnout | Minimum viable day = 1 small win + rest. No guilt. Tomorrow is reset. |
| After frustration | Switch context completely. Quick win in different domain. Return later with fresh eyes. |
| After long session | Movement + nature + disconnect. Minimum 30 min offline before next session. |
| After failure/regression | Acknowledge. Analyze root cause. Document lesson. New session with clean context. |
| After success/flow | Capture what worked (memory/notes). Don't chase the high — rest to sustain. |
