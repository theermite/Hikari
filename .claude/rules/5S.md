# 5S — Clean Workshop, Clear Mind

**Proof state**: 🟢 robust — Toyota TPS, documented industrial method.

> Full source: github.com/theermite/Shinzo · `07-Methode/Regles/5S.md`
> Toyota (TPS) method ported to dev. **A reading grid (philosophy), NEVER a 7th
> manual barrier.**

**Level**: GUIDE — a reading grid, not BLOCKING (see Status).

A messy workshop hides defects; a clean workshop makes them obvious. Ported: a clean
repo surfaces anomalies. 5S adds no new rule — it organizes the existing rules under
an observable angle.

**The 5 pillars**:

| # | Pillar | In dev |
|---|--------|--------|
| S1 | Seiri (sort) | Remove dead code, stale TODOs, unused deps, plaintext secrets, dead branches |
| S2 | Seiton (set in order) | Each thing in its place: coherent structure, naming, Lego library (zero duplication), registry up to date |
| S3 | Seiso (shine) | At every commit: tests green, lint zero, clean UTF-8 encoding, build without warning |
| S4 | Seiketsu (standardize) | One way to do each thing: shared templates, code-enforced hooks, automated conventions |
| S5 | Shitsuke (sustain) | The method does not erode: /audit, /session-end, Kata as single source. Code-enforced > human-enforced |

**5S vs Monozukuri**: Monozukuri = the attitude (who am I when I produce?). 5S = the
observable gestures (what is the state of my workshop?). Both needed.

**Status (decision 2026-06-23)**: 5S stays a reading grid, never a manual barrier.
The real guards are the existing BLOCKING rules (Quality, Conventions, Workflows) and
their hooks; 5S illuminates them, does not double them. The 6-question end-of-session
test = an optional review tool (signal, not automatic penalty).
